현재 작업의 블로커를 기록하고 해결 방향을 탐색합니다.

## 블로커 내용

$ARGUMENTS

## 실행 절차

1. 블로커 내용이 없으면 "블로커 내용을 입력해주세요. 예: `/stuck API 응답 형식이 문서와 달라서 파싱이 안 됨`"라고 안내하고 종료하세요.

2. `.claude/context/current.md` 파일을 읽으세요. 파일이 없으면 "진행 중인 작업이 없습니다."라고 안내하세요.

3. 컨텍스트 파일의 `## 블로커` 섹션에 아래 형식으로 추가하세요:

```
- [{현재 시각}] {블로커 내용}
```

4. frontmatter의 `status`를 `blocked`로, `updated`를 현재 시각으로 변경하세요.

5. 블로커 해결을 위해 아래를 분석하세요:
   - 블로커의 근본 원인
   - 가능한 해결 방향 2~3가지
   - 권장 접근법

6. 분석 결과를 출력하고, `## 다음 행동` 섹션을 권장 접근법으로 업데이트하세요.

7. 상태를 다시 이전 단계로 되돌리세요 (blocked → 이전 status).
