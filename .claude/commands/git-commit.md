로컬 변경사항을 분석해서 Git 커밋을 수행해주세요.

## 절차

1. `git status`와 `git diff`(staged + unstaged 모두)를 실행해서 변경사항을 파악하세요.
2. 변경사항이 없으면 "커밋할 변경사항이 없습니다."라고 알려주세요.
3. 변경사항이 있으면 아래 규칙에 따라 커밋 메시지를 작성하세요.
4. `git add -A`로 모든 변경사항을 스테이징하세요.
5. 작성한 메시지로 커밋하세요.

## 커밋 메시지 규칙

- Conventional Commits 형식 사용: `<type>: <subject>`
- type: `feat`, `fix`, `refactor`, `docs`, `test`, `chore`, `style` 중 선택
- subject: 한국어로 작성, 명령형, 50자 이내
- 본문이 필요한 경우 한 줄 공백 후 추가

인자가 있는 경우(`$ARGUMENTS`): 해당 인자를 커밋 메시지로 사용하세요.

## 주의사항

- `.env`, 시크릿 파일, 대용량 바이너리는 커밋 전 경고하세요.
- 커밋 완료 후 커밋 해시와 메시지를 출력하세요.
